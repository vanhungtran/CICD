CI/CD Fundamentals

Welcome to the world of Continuous Integration and Continuous Deployment! In this first lesson, we'll explore the fundamental concepts that power modern software development.

What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment/Delivery. It's a methodology that automates the software development lifecycle, from code changes to production deployment.

Key Benefits

  • Faster development cycles
  • Reduced manual errors
  • Improved code quality
  • Better team collaboration

Core Concepts

Continuous Integration (CI)

CI is the practice of automatically integrating code changes from multiple contributors into a shared repository several times a day. Each integration triggers automated builds and tests.

Continuous Deployment (CD)

CD extends CI by automatically deploying code changes to production environments after passing all tests and quality checks.

GitLab Setup & Configuration

Now that you understand CI/CD fundamentals, let's set up GitLab and prepare your environment for building pipelines.

Prerequisites

  • GitLab account (free at gitlab.com)
  • Basic Git knowledge
  • Project repository access

Step 1: Create a New Project

  1. Log in to your GitLab account
  2. Click "New Project" or the "+" icon
  3. Choose "Create blank project"
  4. Enter project name and description
  5. Set visibility level (Private recommended for learning)
  6. Initialize repository with README
  7. Click "Create project"

Step 2: Verify Runners

GitLab provides shared runners by default. Let's verify they're available:

  1. Go to Settings ‚Üí CI/CD
  2. Expand the "Runners" section
  3. Ensure at least one runner shows a green circle
